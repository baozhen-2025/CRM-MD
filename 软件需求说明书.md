# 主数据集成管理平台 软件需求说明书

**文档版本**: V1.2
**编写日期**: 2026-03-19
**项目名称**: 一级公司与三级公司主数据同步管理系统

---

## 1. 项目概述

### 1.1 项目背景

三级公司建有主数据管理系统，统一管理组织架构、客商、人员等维度数据。一级公司的各业务系统也存在相同维度的数据。为保证数据一致性，需要将一级公司的数据同步到三级公司主数据平台，并建立数据匹配映射关系。

### 1.2 项目目标

1. 实现一级公司数据到三级公司主数据平台的 T+1 同步
2. 支持用户自定义配置字段映射关系
3. 支持用户自定义配置数据匹配规则
4. 提供人工审核机制处理无法自动匹配的数据
5. 记录完整的数据变更历史，支持追溯和审计

### 1.3 系统范围

| 项目 | 说明 |
|------|------|
| 数据源 | 一级公司各业务系统（通过API对接） |
| 数据目标 | 三级公司主数据管理平台 |
| 数据维度 | 组织架构、客商、人员 |
| 同步方式 | T+1 增量同步 |

### 1.4 系统架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                         数据存储架构                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│   一级公司系统 ──→ 源数据表 ──→ 匹配处理 ──→ 主数据表               │
│                      │                        │                      │
│                      │                        └──→ 变更日志表        │
│                      │                                             │
│                      └──→ 保留原始数据，支持追溯和重新匹配           │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.3 字段映射配置

#### 2.3.1 功能描述

配置一级公司数据表字段与三级公司主数据表字段的对应关系，包括字段名称映射和值映射。

#### 2.3.2 应用场景示例

**场景**：一级公司的客商表结构与三级公司主数据表结构不同

```
一级公司客户表                          三级公司主数据表
┌────────────────────┐                ┌────────────────────┐
│ cust_code          │                │ customer_id        │
│ cust_name          │                │ customer_name      │
│ credit_code        │                │ uscc              │ ← 字段名不同
│ cust_type          │                │ customer_type      │
│ status             │                │ is_active          │ ← 类型也不同
└────────────────────┘                └────────────────────┘
  (status: 1启用/0停用)                  (is_active: true/false)
```

**字段映射配置**：
```
一级字段          三级字段          匹配主键    值映射
────────────────────────────────────────────────────────
cust_code    →   customer_id       ☑         -
cust_name    →   customer_name     □         -
credit_code  →   uscc              □         -
cust_type    →   customer_type     □         {"客户":"1","供应商":"2"}
status       →   is_active         □         {"1":true,"0":false}
```

#### 2.3.3 功能清单

| 编号 | 功能项 | 说明 | 优先级 |
|------|--------|------|--------|
| F2.3.1 | 配置字段映射 | 建立源字段与目标字段的对应关系 | 高 |
| F2.3.2 | 设置匹配主键 | 指定哪些字段作为数据匹配的依据 | 高 |
| F2.3.3 | 配置值映射 | 当字段值需要转换时配置映射关系 | 中 |
| F2.3.4 | 删除字段映射 | 删除已配置的映射关系 | 中 |

#### 2.3.4 数据项

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 数据源 | 关联 | 是 | 选择已配置的数据源 |
| 一级公司字段 | 文本 | 是 | 源系统字段名 |
| 三级公司字段 | 文本 | 是 | 目标系统字段名 |
| 字段类型 | 下拉 | 否 | 字符串/数字/日期/布尔 |
| 作为匹配主键 | 开关 | 否 | 是否用于数据匹配 |
| 值映射配置 | JSON | 否 | 值转换规则，如{"总部":"1","分公司":"2"} |

#### 2.3.5 业务规则

1. 每个数据源至少需要一个匹配主键字段
2. 同一源字段不能重复映射
3. 值映射配置为JSON格式

### 2.4 匹配规则配置

#### 2.4.1 功能描述

配置数据匹配的条件规则，用于判断一级公司数据是否与三级公司主数据为同一条记录。

#### 2.4.2 应用场景示例

**场景**：一级公司推送一条客商数据，需要判断在三级公司是否已存在

```json
一级公司推送数据：
{ "cust_code": "C001", "cust_name": "华为技术有限公司", "credit_code": "91440300A" }
```

**匹配规则配置**：

| 规则名称 | 匹配条件 | 优先级 | 说明 |
|---------|---------|--------|------|
| 编码精确匹配 | cust_code = customer_id | 1 | 编码相同即为同一条 |
| 信用代码匹配 | credit_code = uscc | 2 | 信用代码相同即为同一条 |
| 名称+类型匹配 | cust_name = customer_name AND cust_type = customer_type | 3 | 名称和类型都相同 |

**匹配结果**：
- 匹配到 **1条** → 更新该记录
- 匹配到 **多条** → 进入人工审核
- **无匹配** → 新增记录

#### 2.4.3 功能清单

| 编号 | 功能项 | 说明 | 优先级 |
|------|--------|------|--------|
| F2.4.1 | 新增匹配规则 | 创建新的匹配规则 | 高 |
| F2.4.2 | 配置匹配条件 | 设置字段匹配的条件组合 | 高 |
| F2.4.3 | 设置优先级 | 规则的执行顺序 | 中 |
| F2.4.4 | 启用/禁用规则 | 控制规则的启用状态 | 中 |
| F2.4.5 | 规则版本管理 | 保留规则的历史版本 | 低 |

#### 2.4.4 数据项

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 规则名称 | 文本 | 是 | 如：组织编码精确匹配 |
| 数据源 | 关联 | 是 | 选择数据源 |
| 匹配条件 | 列表 | 是 | 条件组合（见下方说明） |
| 优先级 | 数字 | 否 | 数字越小优先级越高，默认1 |
| 状态 | 开关 | 否 | 启用/禁用 |

**匹配条件结构**：
```
条件1: 一级字段 [比较方式] 三级字段 [逻辑关系]
条件2: 一级字段 [比较方式] 三级字段 [逻辑关系]
...

比较方式: 等于/不等于/包含
逻辑关系: AND/OR
```

#### 2.4.5 业务规则

1. 匹配规则按优先级顺序执行
2. 规则修改后版本号自动+1
3. 同一数据源至少需要一条匹配规则

### 2.5 同步任务管理

#### 2.5.1 功能描述

管理数据同步任务的配置和执行，控制T+1增量同步的调度。

#### 2.5.2 同步流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                        T+1 同步流程                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. 定时任务触发（如每日凌晨2点）                                    │
│         │                                                            │
│         ▼                                                            │
│  2. 调用一级公司API获取增量数据                                      │
│         │                                                            │
│         ▼                                                            │
│  3. 存入源数据表（md_source_xxx）                                    │
│         │                                                            │
│         ▼                                                            │
│  4. 按字段映射规则进行数据转换                                       │
│         │                                                            │
│         ▼                                                            │
│  5. 按匹配规则进行数据匹配                                           │
│         │                                                            │
│         ├──── 匹配成功 ────→ 更新主数据表，记录变更日志              │
│         │                                                            │
│         ├──── 无匹配 ─────→ 新增主数据表，记录变更日志               │
│         │                                                            │
│         └──── 多匹配/失败 → 进入人工审核队列                         │
│         │                                                            │
│         ▼                                                            │
│  6. 生成同步报告，统计处理结果                                       │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

#### 2.5.3 功能清单

| 编号 | 功能项 | 说明 | 优先级 |
|------|--------|------|--------|
| F2.5.1 | 新建同步任务 | 创建数据同步任务 | 高 |
| F2.5.2 | 手动执行任务 | 立即触发一次同步 | 高 |
| F2.5.3 | 查看执行日志 | 查看任务的历史执行记录 | 高 |
| F2.5.4 | 查看执行详情 | 查看单次执行的统计信息 | 中 |
| F2.5.5 | 启用/停用任务 | 控制任务的定时执行 | 中 |

#### 2.5.4 数据项

**同步任务配置**：
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 任务名称 | 文本 | 是 | 如：组织数据同步 |
| 数据源 | 关联 | 是 | 选择数据源 |
| 调度时间 | Cron表达式 | 是 | 如：0 2 * * * 表示每日凌晨2点 |
| 状态 | 开关 | 否 | 启用/停用 |

**同步执行记录**：
| 字段名 | 类型 | 说明 |
|--------|------|------|
| 批次号 | 文本 | 唯一标识，如：SYNC-2024031802001 |
| 执行时间 | 日期时间 | 任务开始执行时间 |
| 拉取总数 | 数字 | 从一级公司拉取的数据条数 |
| 新增记录 | 数字 | 新增到主数据的条数 |
| 更新记录 | 数字 | 更新的数据条数 |
| 待审核 | 数字 | 进入人工审核的数据条数 |
| 执行状态 | 状态 | 成功/部分失败/失败 |

#### 2.5.5 业务规则

1. 定时任务在指定时间自动执行
2. 支持手动触发执行
3. 每次执行生成唯一的批次号
4. 执行完成后记录同步统计信息

### 2.8 变更日志

#### 2.8.1 功能描述

记录主数据表的所有变更历史，支持追溯和审计。

#### 2.8.2 数据更新策略

当一级公司数据发生变更时，采用**更新+变更日志**策略：

| 表 | 更新策略 | 说明 |
|---|---------|------|
| **源数据表** | 更新 | 保留最新源数据，更新同步时间 |
| **主数据表** | 更新 | 保留最新主数据，版本号+1 |
| **变更日志表** | 新增 | 记录每次变更的前后值 |

**更新示例**：

```
T日数据：{ "cust_code": "C001", "cust_name": "华为技术" }
T+1日数据：{ "cust_code": "C001", "cust_name": "华为技术有限公司" }

更新后：
┌────────────────────────────────────────────────────────────────────┐
│ 源数据表：更新为最新值                                              │
│ source_data = {"cust_code":"C001", "cust_name":"华为技术有限公司"}  │
│ last_sync_time = 2024-03-19                                        │
├────────────────────────────────────────────────────────────────────┤
│ 主数据表：更新为最新值                                              │
│ customer_name = "华为技术有限公司"                                  │
│ version = 2                                                        │
├────────────────────────────────────────────────────────────────────┤
│ 变更日志表：新增一条记录                                            │
│ old_value = {"customer_name":"华为技术"}                           │
│ new_value = {"customer_name":"华为技术有限公司"}                    │
│ change_source = "一级公司同步-SYNC-002"                            │
└────────────────────────────────────────────────────────────────────┘
```

#### 2.8.3 功能清单

| 编号 | 功能项 | 说明 | 优先级 |
|------|--------|------|--------|
| F2.8.1 | 查看变更列表 | 显示所有数据变更记录 | 高 |
| F2.8.2 | 筛选查询 | 按维度、操作类型、时间范围筛选 | 中 |
| F2.8.3 | 查看变更详情 | 显示变更前后的数据对比 | 高 |

#### 2.8.4 数据项

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 变更时间 | 日期时间 | 数据变更发生的时间 |
| 数据维度 | 下拉 | 组织架构/客商/人员 |
| 主数据ID | 数字 | 变更的主数据记录ID |
| 数据标识 | 文本 | 变更数据的名称或编码 |
| 操作类型 | 状态 | 新增/修改/删除 |
| 变更前数据 | JSON | 变更前的值（按字段） |
| 变更后数据 | JSON | 变更后的值（按字段） |
| 变更来源 | 文本 | 一级公司同步-批次号/人工审核/手动修改 |

#### 2.8.5 业务规则

1. 所有主数据变更自动记录日志
2. 日志记录不可修改和删除
3. 变更前后数据按字段对比展示
4. 保留至少1年的变更历史
